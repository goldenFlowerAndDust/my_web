      CSS选择器对比：通用兄弟选择器（~）和子选择器（>）

      通用兄弟选择器（~）

      • 定义：选择某个元素之后的所有兄弟元素。
      • 语法：E ~ F，其中 E 是参考元素，F 是 E 之后的所有兄弟元素。
      • 作用范围：选择 E 之后的所有兄弟元素，无论它们的类型。
      • 动态性：如果兄弟元素的数量或顺序发生变化，选择器仍然有效。
      • 代码量：可以一次性选择多个兄弟元素，减少代码量。
      • 继承问题：可能会导致子元素继承样式，需要额外处理。
      • 适用场景：
        • 动态内容，兄弟元素的数量或顺序可能变化。
        • 批量样式，需要对多个兄弟元素应用相同样式。
        • 指定起始点，从某个特定元素开始选择其后的所有兄弟元素。

      子选择器（>）

      • 定义：选择某个父元素的直接子元素。
      • 语法：E > F，其中 E 是父元素，F 是 E 的直接子元素。
      • 作用范围：仅选择 E 的直接子元素，不包括嵌套的子元素。
      • 动态性：如果子元素的结构发生变化，选择器可能需要调整。
      • 代码量：需要逐个指定每个子元素，可能会增加代码量。
      • 继承问题：不会影响嵌套的子元素，避免了继承问题。
      • 适用场景：
        • 精确控制，需要精确选择直接子元素。
        • 避免嵌套影响，需要避免嵌套元素继承样式。
        • 简单直接，需要对直接子元素应用样式。

focus与selector区别

目录——其他知识补充
对比维度 焦点(focus) 选中(selector)
核心定义 标记【当前要操作的元素】——明确"接下来操作对象" 标记【元素内被选定的内容/选项】——明确"选了元素里的什么"
作用对象 整个可操作元素(如按钮、链接、带tabindex的div) 元素内的具体内容/选项(如文本、复选框选项、下拉框选项、图片)
是否依赖"可选项" 不依赖！无内容/选项的元素也能获取焦点(空白按钮、纯链接) 必须依赖！元素必须有可选择的内容/选项(否则无法选中，如空白按钮不能被"选中")
视觉表现 元素外围/自身的"目标提示"：外轮廓(outline)、边框变色、发光、光标(输入框) 元素内部的 “选中标记”：文本蓝底白字、复选框打√、单选框填充、下拉框高亮选项
触发方式 1. 键盘：Tab 切换、Enter 激活；2. 鼠标：单击元素；3. JS：element.focus () 1. 鼠标：拖拽选文本、单击选项；2. 键盘：Ctrl+A 选文本、方向键切选项、空格勾选；3. JS：window.getselector ()
页面同时存在数量 最多 1 个（同一时间只能有一个 “操作目标”） 可以多个（如同时选两段文本、勾选多个复选框）
是否会触发操作 不会！仅标记目标，不执行任何功能（如按钮获焦≠点击，输入框获焦≠输入） 可能触发（如勾选复选框 = 选中并生效，选中文本 = 可复制）
关联 CSS 特性 :focus、:focus-visible、:focus-within（伪类，针对元素） ::selector（伪元素，针对选中文本）、:checked（伪类，针对选中选项）
常见误区 误区：有焦点 = 被选中 / 被点击 → 纠正：焦点只是 “要操作”，不是 “已操作” 误区：被选中 = 有焦点 → 纠正：选中是 “选了内容”，和焦点无关（可无焦点仍选中）
生活化类比 舞台上的 “追光灯”—— 照谁，接下来就给谁递话筒（操作） 舞台上演员手里的 “剧本高亮部分”—— 选中哪段，就表演哪段（内容选择）
典型场景对比 1. 按 Tab 键，按钮出现蓝色外框；2. 单击输入框，出现光标 + 外轮廓 1. 拖拽选中输入框内的文本（蓝底）；2. 单击复选框，出现√；3. 下拉框选中某选项

选中与焦点可以同时存在，它们不互斥
比如：表单一个输入地址的控件，单击它整个控件临时被方框包围(按页面空白部分消失)是焦点，控件内会有输入文本提示(|)为选中

border与outline区别

目录——其他知识补充
对比维度 border(边框) outline(外轮廓)
核心用途 元素的【固定样式】(长期存在，作为元素本身的一部分) 元素的【临时标记】(短期显示，用于状态提示或调试)
空间占用 占用布局空间(会影响元素尺寸、挤压相邻元素) 不占用布局空间(浮于元素之上，不改变页面布局)
样式灵活性 高：可单独设置单条边、支持圆角、可设置不同边的样式/颜色 底：仅支持全包围元素，无法单独设置单条边，不适配圆角(会超出圆角范围)
视觉表现 与元素融为一体，视觉协调，不会重叠其他内容 独立用于元素，可能与相邻元素重叠，视觉上会更“突兀”(适合突出提示)
语义暗示 表示元素的结构边界(如卡片、按钮的物理边框) 表示元素的特殊状态(如：焦点、选中、悬浮、调试标记……)
常用场景 卡片边框、按钮固定边框、输入框默认边框、导航栏分隔线、盒子的结构边界 ：focus焦点提示、开发调试标记、：hover/：active临时高亮、选中状态提示
与其他样式兼容性 兼容border-radius(圆角)、box-shadow(阴影)、可与元素样式无缝配合 不兼容border-radius(圆角处会显示直角轮廓)，与阴影可能叠加冲突
无障碍关联 无直接关联(仅作样式存在) 强关联(常用于：focus焦点可视化、帮助键盘用户识别焦点位置、符合无障碍要求)
默认行为 元素默认无border(需要手动设置) 部分元素(按钮、输入框)获焦点时，浏览器会自带默认outline(可通过outline：none取消)

基本选择器补充——属性选择器

  1. 什么是属性选择器

    目录——基础选择器补充——属性选择器

    属性选择器的本质是：不看类名/ID，只看元素有没有某个属性、或属性值长什么样，就能精准选中它

  2. 属性选择器(存在匹配)：[attr]

    目录——基础选择器补充——属性选择器
    • 语法：[attr]
    • 顾名思义<：只要元素有[attr]这个属性，不管属值是啥，都会被选中
    • 最基础的属性选择器，适合"批量选中有[attr]的元素"
    • 常见场景：选中所有可点击的链接、所有禁用的表单元素
    • [attr]attr可以是任意属性名

  3. 属性选择器(精准匹配):[attr=value]

    目录——基础选择器补充——属性选择器
    • 语法：[attr=value]
    • 顾名思义<：元素必须有[attr]这个属性，且属性值完全等于value(一字不差)，才会被选中
    • 匹配规则：大小写敏感(比如type="Text"≠type="text")
    • 属性值建议加上引号(单/双都可以)，避免特殊字符出错
    • 常见场景：区分不同类型的表单元素(text/password/email)
    • [attr]attr可以是任何属性名，value可以是任何属性值

  4. 属性选择器(开头匹配)：[attr^=value]

    目录——基础选择器补充——属性选择器
    • 语法：[attr^=value]
    • 顾名思义：元素有[attr]这个属性,且以指定字符value开头，就会选中
    • 就是语法精准匹配(=)前加了符号[^],[^]是"脱字符"
    • 常见场景：区分HTTP/HTTPS链接、选中指定文件夹下的资源
    • [attr]attr可以是任何属性名，value可以是任何属性值

  5. 属性选择器(结尾匹配)：[attr$=value]

    目录——基础选择器补充——属性选择器
    • 语法：[attr$=value]
    • 顾名思义：元素有[attr]这个属性，且属性以指定字符value结尾，就会被选中
    • 就是语法精准匹配(=)前加了符号[$]
    • 常见场景：按文件后缀(png/jpg/pdf)区分元素、批量处理同类型资源
    • [attr]attr可以是任何属性名，value可以是任何属性值

  6. 属性选择器(包含匹配)：[attr*=value]

    目录——基础选择器补充——属性选择器
    • 语法：[attr*=value]
    • 顾名思义：元素有[attr]这个属性，且属性包含指定字符value(字符可以在任何位置),就会被选中
    • 就是语法精准匹配(=)前加了符号[*](相当于模糊搜索)
    • 常见场景：批处理含相同关键字的类名/链接，减少重复写样式
    • [attr]attr可以是任何属性名，value可以是任何属性值

  7. 使用属性选择器需要的注意事项

    目录
    • 属性值的引号：建议始终加引号(比如[type="text"])，避免属性值含空格/特殊字符时失效
    • 大小写敏感：大部分浏览器中，属性值匹配区分大小写(如[]≠[])
    • 优先级：属性选择器优先级=类选择器(.btn)，高于元素选择器，低于ID选择器(#btn)
    • 不要过度使用：如果元素已有明确的class/ID,优先用class，属性选择器适合”无类名可复用“的场景。

目录——伪类选择器补充

结构伪类——：root{}

目录——结构伪类
  • 语法：root{}(一般不会再前面添加selector)
  • 定义：匹配文档的根元素(html中通常是<html>标签)，常用于定义全局CSS变量(自定义属性)，方便统一管理样式
  • 注意事项：
    1. :root 是定义全局 CSS 自定义属性（变量） 的最佳载体，其作用域覆盖整个文档，所有元素均可访问这些变量
    2. :root 唯一匹配 HTML 根元素 <html>，优先级高于普通元素选择器
    3. 核心用途是定义全局 CSS 变量，推荐单独使用，禁止冗余组合
    4. 相同样式下，:root 会覆盖<html> 的样式，开发中需注意优先级冲突
    5. 变量名区分大小写（--main-color 和 --Main-Color 是两个不同变量）
    6. 可通过 var(变量名, 默认值) 设置兜底默认值，避免变量未定义时样式异常
    7. 不要滥用 :root 设置非全局样式： :root 适合定义全局通用样式（如变量、全局滚动行为、全局字体），不适合设置局部元素样式（如某个 div 的边距），否则会增加维护成本

  • 语法：selector:read-write{}
  • 定义：匹配可编辑的元素怒(未添加readonly且未禁用的元素)
  • 注意事项：
    1. 与：read-only互为反状态，默认情况下可编辑的表单元素均匹配此选择器

表单专属伪类：placeholder-shown

目录——表单专属伪类选择器
  • 语法：selector:placeholder-shown
  • 定义：选中当前正在显示占位符文本(placeholder属性值)的表单元素
  • 注意事项：
    1. 生效的两个条件
      • 元素设置了 placeholder 属性（如 <input placeholder="请输入..."> 或 <textarea placeholder="请输入...">）
      • 元素处于 “未输入状态”，即用户未输入任何内容，占位符文本可见。
    2. 触发时机：当用户未输入内容时，占位符显示 → 匹配 :placeholder-shown；
      当用户开始输入（即使输入后删除内容，只要占位符消失）→ 不再匹配 :placeholder-shown
    3. 适用元素：仅支持带有 placeholder 属性的表单元素，如 <input>（type 为 text、search、url 等）和 <textarea>
    4. 一般于伪类：（invalid、valid、in-range、out-of-range）搭配适用

目录——选择器的优先级
  • css中，权重用数字衡量
    1. 元素选择器的权重为：1
    2. class选择器的权重为：10
    3. id选择器的权重为：100
    4. 内联选择器的权重为：1000
    5. 综上所述(排名)：行内样式>ID选择器>类选择器>元素选择器
