中文逐行解释（与上面代码对应）

  1. action="/search" → 数据提交到 /search 接口；改为你的后端地址即可。
  2. method="post" → 使用 POST 方式，适合写入/上传；查询可用 GET。
  3. enctype="multipart/form-data" → 允许上传文件；纯文本可省略或留默认。
  4. target="_blank" → 提交后在新标签打开结果；_self 为当前页。
  5. autocomplete="off" → 关闭浏览器自动填充；登录页常设为 off。
  6. novalidate → 禁用原生校验，完全由你或 JS 校验；去掉则浏览器自动拦截空/格式错误。
  7. label + for：点击文字=点击控件，提升可访问性。
  8. required / minlength / maxlength / pattern：前端第一道校验，减轻后端压力。
  9. type="number" 配合 min/max/step：限定数字范围；step="5" 表示每次增减 5。
  10. type="range"：滑块可视化数值；与 number 联动可做双向绑定。
  11. date/datetime-local/month/week/time：原生日期/时间选择器，移动端自动弹出轮盘。
  12. type="color"：原生取色器，返回 #rrggbb 字符串。
  13. type="file" + accept + multiple：文件上传，accept 限制 MIME，multiple 允许多选。
  14. textarea：rows/cols 控制默认大小；maxlength 限制字数。
  15. radio/checkbox：name 相同即为一组；value 为后端实际接收值。
  16. select / select multiple：单选下拉 vs 多选列表；size 控制可见行数。
  17. datalist + list：联想输入，输入框与下拉结合，提升用户体验。
  18. button type 三兄弟：submit=提交表单，reset=清空，button=纯 JS 触发，无默认行为。
  19. hidden：不显示但随表单提交，常放 CSRF、配置 ID 等。
  20. output + for：只读展示结果，for 属性关联参与计算的控件 ID 列表（空格分隔）。
